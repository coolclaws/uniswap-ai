# Documentation Standards

## README Files

Every plugin should have a comprehensive README.md that explains:
- What the plugin does
- How to install and use it
- Available skills and their purposes
- Any configuration required

## Skill Documentation

Each SKILL.md should include:
- YAML frontmatter with name, description, license, and metadata
- Clear usage instructions
- Examples of how to use the skill
- Reference to any external documentation

## CLAUDE.md Maintenance

After making changes to the repository:
1. Update CLAUDE.md with any new architectural decisions
2. Update plugin READMEs if functionality changes
3. Update skill documentation if behavior changes
